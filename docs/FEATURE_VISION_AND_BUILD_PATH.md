# LoreHearth — Feature Vision and Build Path

LoreHearth final target: Android-native character roleplay client with SillyTavern-style workflow parity, common-content compatibility, mobile-first fixes, cloud API mode first, and future offline Campfire Mode.

## v0.1 Hearth Spine

The first milestone is:

Settings → Character Import → Active Character → Chat → Real Model Reply → Persistence.

v0.1 must not pretend future features are done. RAG, TTS, VN mode, group chat, image generation, extensions, and local GGUF inference wait until later phases.

## Final Feature Wishlist

LoreHearth should eventually support:

- Character cards and Lore Vault.
- User personas.
- Scenario and first-message handling.
- Generation presets.
- Cloud provider routing.
- OpenAI-compatible APIs.
- Local offline RP through Campfire Mode.
- Lorebooks / World Info.
- Prompt construction and prompt inspection.
- Context budget management.
- Regex processing.
- Branching and regeneration.
- Group chat.
- RAG/document retrieval.
- TTS.
- Translation.
- Image hooks.
- Backgrounds/sprites / VN presentation.
- Backup/import/export.
- Extension/add-on compatibility through import, conversion, or adapter layers.
- Accessibility and tablet/phone polish.

## Status Key

- Real: user can use it in the app.
- Partial: code exists but the full user flow is incomplete.
- Placeholder: class/screen exists but does not perform the real feature.
- Planned: belongs in the final app but should wait.
